# 力扣hot100-两数之和

[1. 两数之和 - 力扣（LeetCode）](https://leetcode.cn/problems/two-sum/description/?envType=study-plan-v2&envId=top-100-liked)

## 题目：

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** *`target`* 的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

## 题解：

两种方法：一个是暴力解，一个是用哈希表

### 暴力：

通过两层循环遍历数组中的所有可能数对，逐一检查两个数之和是否等于目标值 `target`。一旦找到符合条件的下标对，立即返回。这种方法逻辑简单，但时间复杂度为 O(n²)，在数组规模较大时效率较低。

### 哈希表：

更优的解法是借助哈希表（HashMap）实现**“空间换时间”**。核心思想是遍历数组一次，对于当前元素 `x`，只需在哈希表中查找是否存在 `target - x`。由于哈希表的查找时间复杂度为 O(1)，因此整体复杂度降至 O(n)。在遍历过程中，先将当前元素的下标存入哈希表，以便后续元素查询；但要注意必须先查询再存储，这样可以避免匹配到自身，符合“不能重复使用同一元素”的要求。这种一边遍历一边记录的方式，既保证了结果唯一性，又大幅提升了执行效率，是解决此类查找配对问题的经典模式。

## 代码

```
class Solution {
    public int[] twoSum(int[] nums, int target) {
        /*暴力
        for(int i = 0;i < nums.length;i++){
            for(int j = i + 1;j < nums.length;j++){
                if(nums[i] + nums[j] == target){
                    return new int[]{i,j};
                }
            }
        }*/
        //哈希表hashmap
        Map<Integer,Integer> idx = new HashMap<>();
        for(int i = 0;i < nums.length;i++){
            int x = nums[i];
            //判断哈希表中是否有target - x
            //如果找到了就返回结果
            if(idx.containsKey(target - x)){
                //返回两个数的下标
                return new int[]{idx.get(target - x),i};
            }
            //没有找到就将其存入hash表中，key是当前循环到的数值，value是当前数值的下标
            idx.put(x,i);
        }
        return new int[0];
    }
}
```

# 力扣hot100-三数之和

[15. 三数之和 - 力扣（LeetCode）](https://leetcode.cn/problems/3sum/description/)

## 题目：

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：**答案中不可以包含重复的三元组。

## 题解：

本题的解法是**“排序 + 双指针”**。

解题思路是：先将数组排序，然后固定一个数 `nums[i]`，将问题转化为在剩余的有序区间内寻找两个数，使它们的和等于 `-nums[i]`。为了确保结果不重复，固定数时需要跳过所有相同的值；同时，在双指针移动过程中，每当找到一组满足条件的组合（`nums[i] + nums[left] + nums[right] == 0`）时，也要分别跳过左指针和右指针的相邻重复元素，并同时向内收缩指针。这种方法的巧妙之处在于利用排序后的单调性——当三数之和小于零时，左指针右移以增大和；大于零时，右指针左移以减小和，从而将内层查找从 O(n²) 压缩到 O(n)，整体时间复杂度为 O(n²)。

## 代码

```
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        Arrays.sort(nums);
        List<List<Integer>> ans = new ArrayList<>(); 
        int n = nums.length;
        
        for (int i = 0;i < n - 2;i++) {
            int x = nums[i];
            // 跳过重复的i（注意是 nums[i-1]）
            if (i > 0 && x == nums[i - 1]) {
                continue;
            }
            
            int left = i + 1;
            int right = n - 1;
            while (left < right) {
                //计算三个数的和
                int s = x + nums[left] + nums[right];
                //如果三数之和小于0的话，那么说明nums[left]，太小了，需要将left+1
                if (s<0) {
                    left++;
                }
                //如果三数之和大于0的话，那么说明nums[right]，太大了，需要将right-1
                else if (s>0) {
                    right--;
                } else {
                    //找到一组解
                    ans.add(List.of(x,nums[left],nums[right]));
                    //跳过left重复
                    while (left<right && nums[left] == nums[left + 1]) {
                        left++;
                    }
                    //跳过right重复
                    while (left<right && nums[right] == nums[right - 1]){
                        right--;
                    } 
                    //正常移动指针
                    left++;
                    right--;
                }
            }
        }
        return ans;
    }
}
```
