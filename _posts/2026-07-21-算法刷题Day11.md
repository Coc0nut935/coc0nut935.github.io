# 力扣刷题-[25. K 个一组翻转链表](https://leetcode.cn/problems/reverse-nodes-in-k-group/)

## 题目：

给你链表的头节点 `head` ，每 `k` 个节点一组进行翻转，请你返回修改后的链表。

`k` 是一个正整数，它的值小于或等于链表的长度。如果节点总数不是 `k` 的整数倍，那么请将最后剩余的节点保持原有顺序。

你不能只是单纯的改变节点内部的值，而是需要实际进行节点交换。

**示例 1：**

![img](https://assets.leetcode.com/uploads/2020/10/03/reverse_ex1.jpg)

```
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]
```

## 题解：

使用**分段反转 + 链表拼接**进行解题。我们首先遍历链表计算出总长度，进而确定需要翻转的完整组数。为了简化头节点的处理，引入虚拟头节点 `dummy`，并用指针 `p0` 始终指向当前待翻转组的前驱节点。在每一组中，我们使用经典的“头插法”反转 `k` 个节点：用 `prenode` 和 `curnode` 两个指针逐一遍历组内节点，将每个节点的 `next` 指向前一个节点，从而完成局部逆序。翻转完成后，这一组的原首节点（现尾节点）需要与下一组的头节点连接，而前驱 `p0` 则需要指向新的组头。具体操作是：先保存原组首节点（即 `p0.next`）到临时变量，将其 `next` 指向下一组头（即当前 `curnode`），再将 `p0.next` 指向新头（即 `prenode`），最后将 `p0` 移动为刚翻转完的组尾（即原组首），并重置 `prenode` 为 `null` 以准备下一轮。如此循环直到所有完整组处理完毕，最后返回 `dummy.next` 即为新链表头。

## 代码：

```
class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        //计算链表的长度
        int n = 0;
        for(ListNode curnode = head;curnode != null;curnode = curnode.next){
            n++;
        }
        
        // 2. 虚拟头节点，方便处理头节点变化
        ListNode dummy = new ListNode(0, head);
        //哨兵节点，指向当前要翻转的这一组节点的前驱（即上一组的尾节点）
        ListNode p0 = dummy;
        //指向待翻转区间的前一个节点，最终指向翻转后的新头
        ListNode prenode = null;
        ListNode curnode = head;

        //开始反转链表（先反转链表后链接链表）
        int count = n/k;
        while(count!=0){
            for(int i = 0;i<k;i++){
                ListNode tempnext = curnode.next;
                curnode.next = prenode;
                prenode = curnode;
                curnode = tempnext;
            }

            //保存反转后的链表的尾节点
            ListNode tempnext = p0.next;
            //将尾节点的next指向后面的未反转链表的头节点（开始拼接）
            p0.next.next = curnode;
            p0.next = prenode;
            p0 = tempnext;
            prenode = null;
            count--;
        }
        return dummy.next;
    }
}
```
