# 力扣刷题-[76. 最小覆盖子串](https://leetcode.cn/problems/minimum-window-substring/)

## 题目：

给定两个字符串 `s` 和 `t`，长度分别是 `m` 和 `n`，返回 s 中的 **最短窗口 子串**，使得该子串包含 `t` 中的每一个字符（**包括重复字符**）。如果没有这样的子串，返回空字符串 `""`。

测试用例保证答案唯一。

**示例 1：**

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
```

## 题解：

使用 **滑动窗口（双指针）** 配合字符计数数组。需要维护一个动态窗口，用右指针不断向右扩展以纳入新字符，同时用两个哈希表（或数组）分别记录目标字符串 `t` 中每个字符的需求次数和当前窗口中各字符的出现次数。每当窗口内的有效字符（即属于 `t` 且出现次数达到需求）累计覆盖了 `t` 中所有不同字符时，说明当前窗口是一个可行解，此时尝试移动左指针收缩窗口，并在收缩过程中不断更新最短窗口的起始位置和长度。为了高效判断“是否完全覆盖”，我们用一个计数器 `formed` 记录窗口中已满足需求（即 `window[c] >= need[c]`）的字符种类数。当 `formed` 等于 `t` 中不同字符的种类数时，即可进入收缩阶段。

## 代码：

```java
class Solution {
    public String minWindow(String s, String t) {
        int[] need = new int[128];
        for (char c : t.toCharArray()){
            need[c]++;
        } 
        int[] window = new int[128];
        int left = 0, right = 0;
        //formed 为窗口中已匹配的字符总数（不重复计算）
        int formed = 0;
        int required = t.length();
        //{长度, 左, 右}
        int[] ans = {-1, 0, 0}; 
        
        while (right < s.length()) {
            char c = s.charAt(right++);
            //累计有效匹配数
            if (need[c] > 0) {
                window[c]++;
                if (window[c] <= need[c]) formed++; 
            }
            while (formed == required) {
                if (ans[0] == -1 || right - left < ans[0]) {
                    ans[0] = right - left;
                    ans[1] = left;
                    ans[2] = right;
                }
                char d = s.charAt(left++);
                if (need[d] > 0) {
                    if (window[d] == need[d]) formed--; 
                    window[d]--;
                }
            }
        }
        return ans[0] == -1 ? "" : s.substring(ans[1], ans[2]);
    }
}
```
